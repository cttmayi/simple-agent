# Debug Skill

Use this skill when encountering bugs, test failures, or unexpected behavior.

## Debugging Workflow

1. **Understand the Problem**
   - What should happen?
   - What actually happens?
   - Reproduction steps?
   - Error messages or stack traces?

2. **Gather Information**
   - Read relevant code files
   - Check logs
   - Examine test cases
   - Verify assumptions

3. **Form Hypotheses**
   - List possible causes
   - Prioritize by likelihood
   - Test each hypothesis

4. **Implement Fix**
   - Make minimal changes
   - Verify fix works
   - Add tests if needed
   - Document the root cause

5. **Prevent Recurrence**
   - Consider edge cases
   - Update documentation
   - Add to checklist

## Debugging Commands

Use these tools to investigate:
- `read` - Read source code
- `bash` - Run commands/tests
- `grep` - Search for patterns

## Questions to Ask

- Did this work before?
- What changed recently?
- Is this environment-specific?
- Can I reproduce it reliably?
